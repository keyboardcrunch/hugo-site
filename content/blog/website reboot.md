**Website Migration: Back to Basics**

Over the years, my personal website has undergone multiple iterations. However, content migration between versions has never been a strength of mine. In recent times, maintaining the site has simply lost priority amid other responsibilities.

The previous version was built with Deno and deployed on Deno Deploy Classic. With that platform now deprecated, I have opted for a more sustainable approach: a Hugo-based static site hosted on GitHub Pages.

This shift prioritizes simplicity and long-term maintainability over more dynamic but demanding setups. While it reflects a degree of fatigue with ongoing upkeep, it ensures the site remains accessible without unnecessary overhead.

Expect updates to be less frequent, but the foundation should now prove far more reliable.
